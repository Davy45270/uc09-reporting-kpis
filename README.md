# UC09 Reporting & KPI

## Expression de besoin fonctionnelle
- **Objectifs** : dashboards multi‑niveaux, KPIs standard, reporting exécutif automatisé.
- **Acteurs** : direction, PMO, managers, finance.
- **Flux clés** : collecte données → agrégation → visualisation → diffusion → revue périodique.
- **Exceptions** : ajustements manuels documentés; data quality incidents.
- **Critères d’acceptation** : KPIs définis, data fraîche <24h, exports/presentations.

## Analyse ROI
- **Gains** : réduction du reporting manuel, meilleure prise de décision, transparence.
- **Coûts** : modélisation données, connecteurs BI, gouvernance.
- **KPI cibles** : temps de préparation report, taux d’utilisation des dashboards, fraîcheur des données.
